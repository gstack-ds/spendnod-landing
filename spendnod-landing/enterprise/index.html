FAQ

Common questions

Is SpendNod a payment processor?

No. SpendNod is an authorization layer. We never process, hold, or transfer funds. We provide the decision (approved, pending, denied) and the audit trail. Your existing payment infrastructure handles the actual transaction.

We're a US company. Why do we need this?

Three reasons. First, liability - if your AI agent overspends on a client account, you need proof that it was operating within authorized parameters. SpendNod provides that proof. Second, if you have any EU customers or operations, the EU AI Act applies to you (enforcement August 2, 2026). Third, insurers and auditors are starting to ask about AI governance controls. Having SpendNod is a checkbox on applications and audit reviews.

How long does integration take?

For MCP-compatible agents: add one URL to your config file. For REST API integration: one API call before each transaction. Most teams are live within a day, not weeks.

What happens if SpendNod goes down?

If SpendNod is unreachable, your agent receives a timeout error. The agent should be configured to fail closed - meaning it does not proceed with the transaction. This is the safe default and is required for compliance.

Can we self-host?

Yes, on the Enterprise plan. We provide deployment packages for your own infrastructure. Your data never leaves your environment. Contact sales for details.

What agent frameworks are supported?

SpendNod works with any agent that can make an HTTP call or connect via MCP. This includes Claude, ChatGPT, LangChain, CrewAI, AutoGen, Microsoft Agent Framework, and custom agents. If it can call a URL, it works with SpendNod.
